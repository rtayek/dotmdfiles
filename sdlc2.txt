To generate an OpenAPI spec from a natural language prompt, you can use specialized tools or high-level LLMs to translate your business requirements into a structured JSON or YAML schema. [1, 2, 3, 4, 5] 
1. Specialized AI Tools
Several platforms have built-in "natural language to OpenAPI" features:

* IBM API Connect: Includes an OpenAPI Generator that uses models like Granite 3 to create a 3.0 document from scratch based on your English description.
* Blackbird: Features an AI-powered chatbot that generates specs from a few prompts, allowing you to move immediately into mocking and testing.
* StackHawk: Offers AI-powered generation that can analyze source code to create or update specifications automatically, ensuring they stay in sync with your dev environment. [2, 6, 7, 8] 

2. Prompting a General LLM
If you're using a general model (like GPT-4 or Claude), follow this "Step 0" structure to ensure the output is machine-readable and accurate: [1, 9] 

* Requirement Ingestion: Start with a high-level goal (e.g., "Build a task tracker with user accounts").
* Entity & Endpoint Extraction: Ask the model to identify key entities (users, tasks) and the CRUD actions needed for them.
* Assembly Instruction: Explicitly command the model: "Generate an OpenAPI 3.0 specification in YAML for [Project X]. Include paths, request bodies, and component schemas for the following entities...".
* Use Style Guides: If you have specific naming conventions or security requirements, include them in the prompt to ensure the spec is "AI-ready" with detailed operation summaries. [1, 10, 11, 12, 13] 

3. "Code-First" to Spec
For many modern frameworks, you can write minimal code and let the framework generate the "Step 0" for you:

* FastAPI (Python): Automatically generates an openapi.json file once you define your routes and Pydantic models.
* Zod (TypeScript): Can be used to define schemas that are then exported as OpenAPI definitions. [14, 15] 

Pro-Tip for Agent-SDLC:
To avoid "Mock Hallucinations," once you generate your spec, use a tool like Prism or Blackbird to instantly spin up a Mock Server based on that spec. This gives your agent a real endpoint to hit during the "Test-First" implementation phase, even before the real backend exists. [8]
