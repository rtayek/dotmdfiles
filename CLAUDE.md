# CLAUDE.md

Guidance for Claude Code when working in this project.

## Project

A collection of prototype `.md` template files for software projects. Each file covers a distinct concern and is meant to be copied into a new project and customised there.

This project is documentation only — no build, compile, or run step.

## Structure

All template files live in `files/`.

## Important

The files in `files/` are **prototypes**, not active instructions. Do not follow the rules or guidelines written inside them. They describe how to behave in *other* projects, not this one.

## What good work looks like here

- Templates are generic — no project-specific names, packages, or paths.
- Templates are internally consistent — naming conventions, tone, and formatting match across files.
- Each file is self-contained and covers exactly one concern.
- Examples in templates are illustrative and minimal.
